# Specification: Standardized Long-Audio ASR Benchmarking (Earnings-22)

## Overview
This track implements a standardized benchmarking suite to evaluate the `gemini-transcribe` engine against the **Earnings-22** dataset, the current gold standard for long-form ASR. Unlike short-snippet tests, this benchmark evaluates the entire system—including VAD, Global Memory, and Sliding Window Transcription—on real-world earnings calls lasting 15 to 90 minutes.

## Functional Requirements
1. **Benchmark Orchestrator (`scripts/benchmark.py`)**:
   - **Data Source**: Stream the `full` configuration of `distil-whisper/earnings22` from Hugging Face.
   - **Sampling**: Randomly select **3 unique long-form audio files** from the `test` split.
   - **Workflow Integration**: For each file, invoke the full `STTGraph` workflow defined in `app/graph.py`.
   - **Multi-Model Run**: 
     - Run 1: `gemini-3-flash-preview` (Local Proxy).
     - Run 2: `gemini-3.1-flash-lite-preview` (Official API).
2. **Evaluation Metrics**:
   - Calculate Word Error Rate (WER) using the `evaluate` library.
   - Compare the concatenated transcript from our sliding window workflow against the official ground truth transcript provided in the dataset.
3. **Differentiated Audio Preprocessing**:
   - **Global Pass**: Implement ultra-low bitrate compression (**Opus 32kbps**) for the full audio file to stay within 100MB `inlineData` limits for files up to 4-5 hours.
   - **Transcription Pass**: Maintain high fidelity for individual segments (**WAV** or **Opus 128kbps**) to ensure maximum STT accuracy.
4. **Configurable Sliding Context Window**:
   - Implement a logic to inject a user-defined number of previous chunks as context into each transcription prompt.
   - **Default**: Previous 5 chunks.
   - **Goal**: Prevent context "explosion" while maintaining semantic continuity over long transcripts.
5. **Feasibility Validation (2h+ Audio)**:
   - Verify that the 2-hour audio file can be processed using the dual-compression strategy and the sliding window logic.
6. **Reporting**:
   - Generate comparative metrics showing the effect of different context window sizes on WER.

## Non-Functional Requirements
- **Stability**: The benchmark must handle the full processing of 1-hour audio files without state corruption or crash.
- **Resource Management**: Temporary audio chunks generated by VAD must be cleaned up after each model run to save disk space.

## Acceptance Criteria
- 3 random Earnings-22 calls are fully processed by both models.
- Final WER scores are calculated for each model across all 3 files.
- A summary report compares our results with the Gemini 1.5 Pro (2.8%) and Whisper (11.2%) benchmarks.
